How the agent works

STEP 01
Seed the fabric
Spider nodes sign mass-balanced reactions (ChemistryKnitweb) and weave them into the P2P web. Each record gets a deterministic CID via canonical_cid(). Propagated over the 5mart.ml relay.
STEP 02
Lens retrieval
interpret.retrieve() runs a subscription-gated graph walk from query seeds, returning a frozen CandidateSet with provenance roots — without ever touching raw model weights.
STEP 03
Token distillation
The Lens layer strips each record to its relational skeleton (equation + kind). Full JSON: 1 066 tokens. Distilled: 74 tokens. 93% eliminated before the LLM call — saving $0.003 per request at GPT-4 pricing.
STEP 04
Code → GitHub
The agent synthesises a typed Python module (80 lines) from the recovered reactions and pushes it to GitHub via the Contents REST API — no git binary required. Set GITHUB_TOKEN + GITHUB_REPO to push for real.

Run it yourself

Clone, install, and run the chemistry agent demo in three commands. No API keys needed for the dry-run. Set GITHUB_TOKEN to push real code. 

# 1  clone
git clone https://github.com/Knitweb/pulse && cd pulse

# 2  install (venv recommended)
pip install cryptography

# 3  run the Chemistry Lens Agent demo
PYTHONPATH=src python examples/chemistry_lens_agent_demo.py

# → seeds 4 reactions, retrieves via Lens, saves 93% tokens, synthesises 80 lines
# → set GITHUB_TOKEN=<pat> GITHUB_REPO=owner/repo to push generated code for real
