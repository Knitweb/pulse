registered worlds

One vote per person per world.

max_vote_supply = registered + expected_births — anchored to real population. No operator can mint governance weight out of thin air. Float analytics reads the registry; only integers cross into the knitweb settlement layer.

two ends of the fabric

Quantum Forge on one side, Vank Mesh Exchange on the other.

The same WNW record fabric can hold code, run results, and markets. At the research end it becomes a GitHub-like forge for quantum circuit code with MLflow-style experiment lineage. At the Vank end it becomes a non-custodial P2P exchange: Mt. Gox/Bitcoin.de/Intersango market familiarity without a central wallet or hosted orderbook as trust root.

quantum forge

Code, circuits, runs.

Gither becomes the code-forge layer for quantum work: repositories, issues, reviews, signed refs, circuit manifests, and reproducible run records. MLflow-like tracking lives in the fabric rather than a single experiment server.

seam contracts

Every crossing is explicit.

The seam table is the public contract between the float analytics layer (vang / WorldDAO) and the knitweb integer value path. SettlementOrder.amount must be int — TypeError on float.

#SeamDirectionOwnerCrossing rule
1Registry (identity)knitweb → analytics knitweb Subject digest only — raw PII never stored
2Marks ingestknitweb → analytics both Integer marks become float analytics at explicit divisor
3World binding (WorldDAO)analytics ⇄ knitweb vang cap() reads knitweb registry; membership additive-only
4Settlement orderanalytics → knitweb knitweb float → int at quantize(); knitweb never imports float layer
5Dual-signed Knitknitweb internal knitweb nonce + network id + balance guards all enforced in ledger/knit.py

self-hosted node

Run your spider. No hosting provider.

A WNW spider is a single Python process: HTTP gateway + WebGPU worker + P2P relay client. One command installs it and registers it on the 5mart.ml relay fabric.

# Install Vulkan (Ubuntu/Debian) — required for WebGPU on Linux sudo apt install -y vulkan-tools mesa-vulkan-drivers libvulkan1
# Clone + install git clone https://github.com/Knitweb/wnw && cd wnw python3 install.py --yes
# Or run directly (no service) python3 infra/node/spider.py --host 0.0.0.0 --port 8090
# Service management systemctl --user status wnw-spider journalctl --user -u wnw-spider -f
# macOS 10.14+ — Metal is always available git clone https://github.com/Knitweb/wnw && cd wnw python3 install.py --yes
# Or run directly python3 infra/node/spider.py --port 8090
# Manage launchd agent launchctl list | grep knitweb launchctl stop com.knitweb.wnw tail -f /tmp/wnw-spider.log
# Run PowerShell as Administrator git clone https://github.com/Knitweb/wnw; cd wnw python install.py --yes
# Or install Windows Service manually (DX12/Vulkan) powershell -ExecutionPolicy Bypass -File infra\service\Install-WNWService.ps1
# Manage service sc query WNWSpider sc start WNWSpider Get-Content -Wait C:\ProgramData\knitweb\wnw-spider.log
