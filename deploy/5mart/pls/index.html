PLS — the Pulse pay-token

PLS pays for verified useful work on the Knitweb: GPU compute, quantum path sampling, validation, relay and storage. No premine, no admin mint — every PLS in existence was created against a verified proof of work someone actually paid for. Balances are integers; every client agrees byte-for-byte.

Launch faucet — 150 places per country

Each country gets 150 granted faucet places; everyone after that joins a fair FIFO waitlist. One address per person. No personal data is stored — only your pls1 address, country code and a timestamp.

How the faucet stays honest

Faucet coins are reservations settled by the treasury against real worked value — the no-premine rule holds even for the faucet. Per-country caps spread the launch instead of concentrating it. Phase 2 upgrades country checks from self-declaration to zero-PII personhood anchors (vBank, eIDAS/EUDI seam) before any real value moves. Full design: docs/DUAL_COIN_IPO_PLAN.md in Knitweb/pulse.
